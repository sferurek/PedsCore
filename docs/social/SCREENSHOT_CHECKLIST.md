# Screenshot Checklist

Recommended screenshots for public launch posts and release assets.

Do not include patient-identifiable information or real clinical cases. Use only empty states, synthetic values or clearly non-clinical demo values.

## Web App

- Home hero with metrics and CTAs.
- Tools page with status counters and filters.
- WHO Growth form before entering values.
- WHO Growth chart with synthetic demo values.
- Evidence page showing implementation status explanation.
- Disclaimer page or footer showing no clinical data storage and license notes.

## GitHub

- GitHub README top section with badges.
- Issue template chooser.
- Release page after manual release creation.

## QA Before Sharing Screenshots

- Confirm mobile layout has no horizontal overflow outside chart containers.
- Confirm WHO Growth chart labels are readable.
- Confirm screenshots do not imply diagnosis or treatment advice.
- Confirm no browser autofill or personal account data is visible.
- Crop browser chrome if it distracts from the content.
